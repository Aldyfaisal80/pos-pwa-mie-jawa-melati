Keterangan — WF-03-D

Antarmuka Manajemen Produk pada desktop menggunakan pola desain Data Table untuk penyajian informasi persediaan (inventory) secara terstruktur. Struktur tabular linier memfasilitasi komparasi harga dan status ketersediaan secara sekilas, mematuhi prinsip keterbacaan (scannability). Elemen pengelolaan tingkat tinggi seperti "Tambah Produk" dan "Kelola Kategori" diposisikan secara prominen pada tajuk (header) sesuai alur baca pola-F (F-shaped reading pattern).

Keterangan — WF-03-M

Antarmuka Manajemen Produk pada mobile menerapkan pola table-list yang mengoptimalkan kepadatan informasi pada layar sempit. Berbeda dengan desain kartu (card layout) yang boros ruang vertikal, pendekatan baris tabel tipis memungkinkan lebih banyak produk terlihat sekaligus tanpa pengguliran (above-the-fold content). Ikon kategori berwarna-beda di kiri nama produk membantu identifikasi visual cepat. Tombol edit (biru) dan hapus (merah) diposisikan di ujung kanan mengikuti pola destructive action separation. Navigasi tambah produk dipindahkan ke tombol + di sub-header agar dapat dijangkau tanpa ibu jari harus menjangkau pojok layar (Fitts's Law).

Keterangan — WF-03-MF

Modal Form Produk mereduksi hambatan masukan (input friction) lewat arsitektur informasi linear. Pengurutan data dimulai dari hierarki tertinggi yakni representasi grafis (foto), disusul identitas (nama, kategori), bernuara ke atribut transaksional (harga). Penggantian input kuantitatif (total stok) dengan komponen status biner (Toggle Aktif/Inaktif) menandai abstraksi kompleksitas sistem inventaris demi menyederhanakan beban manajemen kasir UMKM.
