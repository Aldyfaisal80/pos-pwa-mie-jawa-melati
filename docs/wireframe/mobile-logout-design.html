Mobile Logout — 3 Design Options

Pilih pendekatan terbaik untuk UX aplikasi POS kamu

Masalah: User mobile tidak bisa logout dari aplikasi
★ Recommended Low Effort · 1 file
Option A — Logout Tab (Compact)
Tambah tombol logout kecil di paling kanan bottom tab bar, dipisahkan dengan garis vertikal. Tab navigasi tetap flex-1, logout flex-none 48px sehingga tidak memenuhi ruang.

Pros

  • 1 file saja diubah
  • Selalu terlihat, mudah ditemukan
  • Pola konsisten dengan sidebar desktop
  • Zero extra components

Cons

  • Tab bar lebih sempit (~48px/tab)
  • Logout berdampingan dengan navigasi
  • Masih > 44px touch target (a11y OK)

Alternative Medium Effort · 2–3 files
Option B — Avatar Dropdown (Top Header)
Tambahkan global mobile header dengan tombol avatar di kanan atas. Klik avatar membuka dropdown kecil berisi info user + tombol logout. Polished, seperti aplikasi Gmail / Google Drive mobile.

Pros

  • Tab bar tidak bertambah item
  • Pola sangat familiar (Google apps)
  • Bisa tambah fitur lain (profil, tema)
  • Visual lebih kaya / informative

Cons

  • Perlu modifikasi PageContainer / layout
  • Mengurangi ruang konten (~48px)
  • Multi-file change

Option C — User Sheet dari Tab Pengaturan
Tab "Pengaturan" berubah menjadi "Akun". Tap membuka bottom sheet dengan info user, shortcut ke pengaturan, dan tombol logout. Tab bar tidak bertambah dan tidak ada header baru. Pattern: seperti Spotify, Tokopedia.

Pros

  • Tab bar tetap 5 item, bersih
  • Efek "wow" — UX premium
  • Grouping logis: akun + pengaturan + logout
  • Tidak mengurangi ruang konten

Cons

  • Logout tidak langsung terlihat
  • Perlu komponen Sheet baru
  • User harus tap 2x untuk logout
