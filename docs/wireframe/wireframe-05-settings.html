Keterangan — WF-05-AS

AccountSheet Mobile mengimplementasikan pola bottom sheet overlay sebagai mekanisme manajemen akun kontekstual pada antarmuka bergerak. Pendekatan ini mempertahankan kebersihan tab bar (lima tab) sambil menyediakan akses cepat ke fungsi manajemen profil dan terminasi sesi. Elemen destruktif "Keluar dari Aplikasi" diberi penanda warna merah (danger color) untuk membedakannya secara visual dari aksi navigasi reguler, sesuai prinsip hierarki tindakan (action hierarchy) dalam desain antarmuka modern.

IDKomponenFungsiInteraksi

Keterangan — WF-05-AC

Panel Akun diimplementasikan sebagai komponen Dialog Modal pada Desktop yang diakses melalui Dropdown Sidebar, dan di-render di dalam Drawer pada state Edit Profil di Mobile. Tata letak mematuhi prinsip pengelompokan visual gestalt, memisahkan domain identifikasi (Email dan Nama Tampilan) dari domain otentikasi kredensial (Ganti Password). Separasi ini meminimalkan risiko perubahan yang tidak disengaja pada data sensitif. Field Email dibuat read-only sesuai kebijakan integrasi identitas dari sisi Auth provider.

IDKomponenFungsiInteraksi
A1Email Akun (disabled)Menampilkan email login yang terdaftar di Supabase Auth. Field tidak bisa diedit dari aplikasi.Read-only — tidak ada interaksi
B1Form Nama TampilanInput untuk memperbarui display_name di user_metadata Supabase. Tombol "Simpan Nama" hanya aktif jika nilai berubah (isDirty).Input → Simpan Nama → Supabase updateUser()
C1Form Ganti Password3 field: Password Saat Ini (verifikasi), Password Baru, Konfirmasi Password Baru. Semua field memiliki toggle show/hide (ikon mata). Minimal 8 karakter.Isi 3 field → Ubah Password → re-auth + updateUser(); form di-reset jika berhasil
