Keterangan — WF-04-D

Antarmuka Laporan Penjualan berpusat pada komponen Data Grid historis yang dikombinasikan dengan indikator performa utama (Key Performance Indicators/KPI). Tata letak ini mendukung kapabilitas pemrosesan analitik secara mandiri lewat fitur rentang tanggal (date-range filter). Penempatan Summary Strip di atas tabel memberikan rasio wawasan-data (data-to-insight ratio) yang tinggi; setiap kartu KPI dilengkapi ikon berwarna (amber untuk pendapatan, biru untuk transaksi, hijau untuk rata-rata) untuk identifikasi kategori yang cepat. Setiap baris tabel menyediakan dua aksi: lihat detail (Eye) dan hapus transaksi (Trash) dengan konfirmasi dialog.

IDKomponenFungsiInteraksi
A1Filter BarFilter periode (tanggal mulai-akhir), metode pembayaran, pencarian no. nota, dan per-page. Export CSV (hijau) semua data yang difilter.Isi filter → tabel otomatis diperbarui (live filtering)
B1Kartu KPITotal Pendapatan (amber), Jumlah Transaksi (biru), Rata-rata/Transaksi (hijau) — ikon + nilai + label periodeAuto-update saat filter berubah
C1Tabel TransaksiKolom: No. Nota (monospace), Waktu (tanggal + jam), Metode (badge), Total (bold), Items, AksiKlik Eye → Modal Detail | Klik Trash → Dialog konfirmasi hapus

Keterangan — WF-04-M

Tampilan mobile Laporan mengadaptasi tata letak desktop dengan pendekatan responsif vertikal. Filter bar diperluas menjadi dua baris: baris pertama untuk pencarian no. nota, baris kedua untuk rentang tanggal dan metode pembayaran. Kartu KPI menggunakan tata letak satu-kolom penuh untuk pendapatan dan dua-kolom untuk metrik transaksi. Daftar transaksi diimplementasikan sebagai tabel responsif dengan kolom Metode tersembunyi di mobile (hanya terlihat di tablet/desktop).

Keterangan — WF-04-MD

Modal Detail Transaksi menerapkan prinsip pengungkapan progresif (progressive disclosure) dengan menyembunyikan atomisitas pesanan dari tabel utama untuk mencegah kepadatan komponen visual. Informasi metadata (no. nota, waktu, metode) ditampilkan dalam grid dua kolom tanpa field kasir, sesuai skema data yang tersedia. Daftar item menggunakan tabel dua kolom (Item | Harga) untuk konsistensi visual. Area Cetak Struk memodelkan komponen PrinterActionButtons: tombol Sambungkan Printer digunakan untuk inisiasi koneksi Bluetooth, sementara Cetak Ulang Struk aktif setelah koneksi terbentuk.
