Keterangan — WF-02-D

Antarmuka Kasir pada desktop dirancang dengan struktur tata letak kolom ganda asimetris (katalog produk dan keranjang pesanan). Pendekatan ini memungkinkan pemrosesan transaksi berkecepatan tinggi tanpa perpindahan konteks halaman (context switching). Filter kategori berbasis pill memfasilitasi penemuan item secara efisien sesuai Hukum Hick (Hick's Law). Detail pesanan aktif selalu terlihat secara utuh, memberikan konfirmasi seketika terhadap setiap modifikasi pesanan.

IDKomponenFungsiInteraksi
A1Tab Kategori (Pill)Filter katalog per kategori yang dibuat di Menu Produk. Rounded-full style. Tidak ada search bar.Klik pill → filter produk
B1Katalog ProdukGrid 4 kolom desktop (xl breakpoint). Background warna per kategori + icon SVG. Klik card → tambah ke keranjang.Klik card → tambah ke keranjang
C1Panel Pesanan AktifDaftar item + kontrol ±qty per baris + link "+ Catatan" + orange note badge jika ada catatan. Trash icon button (kanan atas) untuk kosongkan semua.Klik ± → ubah qty; klik Catatan → NoteModal; Trash → konfirmasi kosongkan
C2Footer KeranjangSubtotal, Pajak (0%), Total Tagihan. Tombol "Bayar Sekarang" → CheckoutModal. Disabled jika keranjang kosong.Klik Bayar → modal checkout

Keterangan — WF-02-M

Adaptasi modul Kasir pada layar mobile menggunakan pendekatan tampilan berpusat pada konteks tunggal (single-context focus), di mana visibilitas diutamakan pada penelusuran katalog. Daftar produk dikondensasi menjadi grid dua kolom, dan panel keranjang direpresentasikan melalui elemen aksi melayang (Floating Action Button) persisten di bagian bawah layar. Transisi dari pemilihan item menuju penyelesaian transaksi dirancang agar mulus, mengurangi beban kognitif pengguna di tengah lingkungan bertempo cepat.

IDKomponenFungsiInteraksi
B1Tab Kategori (Pill)Rounded-full pills lebih kecil untuk mobile. Overflow scroll jika banyak kategori.Klik pill → filter produk
C1Grid 2 Kolom2 kolom di mobile (vs 3 di desktop). Card lebih besar untuk touch. Background grayscale + icon SVG.Klik card → tambah ke keranjang
D1Floating Checkout ButtonPill button fixed di atas bottom nav. Tampil hanya jika keranjang tidak kosong. Berisi qty + total + Bayar.Klik → buka CheckoutModal

Keterangan — WF-02-MC

Implementasi Modal Checkout memanfaatkan pola Overlays bersyarat untuk membatasi interaksi hanya pada alur pembayaran. Isolasi fokus tugas (task isolation) mencegah ambiguitas interaksi akibat interupsi fungsional lain. Penyajian rincian tagihan dikelompokkan secara hierarkis, dengan metode pembayaran diposisikan secara prominen agar proses penyelesaian dapat dieksekusi secara intuitif, meminimalisasi potensi cacat pengguna (user error) dalam penanganan transaksi.

IDKomponenFungsiInteraksi
A1Rincian PesananDaftar item dengan qty badge, harga, link +Catatan, dan stepper ±qty. Header "Rincian Pesanan".Klik ± → ubah qty; klik +Catatan → NoteModal
B1Total TagihanAngka total besar di tengah. Update realtime saat qty item diubah.—
C1Metode PembayaranToggle 3 metode: Tunai / QRIS / Transfer. Jika QRIS → tampilkan QR placeholder. Jika Tunai → input nominal + 4 quick-cash button (15K/20K/50K/100K).Klik → ganti metode
D1Selesaikan TransaksiTombol CTA tunggal. Disabled jika keranjang kosong atau nominal tunai kurang. Spinner loading saat proses.Klik → proses transaksi → Modal Struk

Keterangan — WF-02-MR

Modal Struk berfungsi sebagai validasi akhir pasca-transaksi (post-transaction validation). Desain antarmukanya mereplikasi batasan visibilitas fisik dari format rekam cetak (thermal print) untuk memastikan konsistensi pratinjau digitasl empiris. Seruan bertindak utama (primary call-to-action) "Cetak Struk" ditonjolkan untuk sinkronisasi perangkat keras eksternal, memastikan siklus tutup transaksi diakhiri secara sistematis.

Keterangan — WF-02-MN

Modal Catatan memfasilitasi kustomisasi item secara individual sebagai penambahan metadata independen (tanpa memodifikasi entitas basis objek terkait). Fitur pembeda Tag Groups beroperasi sebagai sistem input terstruktur berkecepatan tinggi, sementara teks bebas mencakup skenario transaksi insidentil. Paradigma umpan balik langsung (instant visual feedback) diadopsi lewat komponen Pratinjau agar operator mudah memvalidasi anotasi spesifik pesanan.
