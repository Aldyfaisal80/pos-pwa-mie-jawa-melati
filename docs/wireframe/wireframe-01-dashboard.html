Keterangan Komponen — WF-01-H-D (Beranda Desktop)

Antarmuka Beranda (Home Hub) dirancang sebagai pusat akses cepat (Quick-Access Hub) yang menggantikan tampilan analitik di halaman root (/). Pendekatan ini mengutamakan kecepatan operasional kasir dengan menempatkan tautan transaksi di posisi paling terjangkau. Komponen GreetingBanner mengimplementasikan konteks temporal dan status konektivitas, sementara QuickActionGrid menerapkan prinsip Hick's Law untuk meminimalkan waktu pengambilan keputusan melalui pembatasan pilihan aksi yang terstruktur.

IDKomponenFungsiInteraksi
A1GreetingBannerGradient card dengan sapaan dinamis (Pagi/Siang/Sore/Malam), nama toko dari API, tanggal/waktu real-time, badge status Online/Offline dari navigator.onLine.Pasif — update otomatis setiap menit
B1MiniStats (2 kartu)Omzet Hari Ini (Rp) dan Total Transaksi hari ini. Data dari tRPC query transaction.getDashboardStats.Pasif, data live dari server
C1QuickActionGrid4 action card vertikal. Card pertama (Mulai Kasir) adalah primary CTA dengan warna brand. 3 card lainnya secondary. Masing-masing navigasi ke: /pos, /products, /reports, /dashboard.Klik/Tap → navigasi ke halaman tujuan

Keterangan Komponen — WF-01-H-M (Beranda Mobile)

Versi mobile Beranda mempertahankan semua komponen Home Hub dalam layout satu kolom yang dioptimalkan untuk layar sempit. GreetingBanner dikompresi namun tetap mempertahankan hierarki informasi. QuickActionGrid menjadi daftar vertikal penuh yang memudahkan interaksi jempol (thumb-reach). Navigasi berpindah ke Bottom Tab Bar 5 tab dengan tab Beranda aktif di posisi pertama, menggantikan Dashboard yang sebelumnya menempati posisi aktif tersebut.

IDKomponenFungsiInteraksi
A1GreetingBanner MobileVersi kompak dengan sapaan, nama toko, dan timestamp. Badge Online/Offline di kanan header.Pasif, update tiap menit
B1MiniStats (2×1)Grid 2 kolom: Omzet dan Transaksi hari ini. Label diperpendek untuk hemat ruang.Pasif, data live
C1QuickActionGrid Mobile4 action card dalam 1 kolom vertikal. Card "Mulai Kasir" berwarna brand (primary). Chevron di kanan tiap card.Tap → navigasi halaman
D1Bottom Tab BarNavigasi 5 tab. Tab Beranda aktif di posisi pertama. Urutan: Beranda, Kasir, Produk, Laporan, Akun. Tab "Akun" membuka Account Bottom Sheet (bukan halaman).Tap tab → pindah halaman; Tap Akun → buka bottom sheet

Keterangan Komponen — WF-01-D (Desktop)

Antarmuka Dashboard utama dirancang menggunakan pendekatan tata letak berbasis grid untuk mengoptimalkan ruang visual pada layar resolusi tinggi (desktop). Modul ini mengedepankan prinsip keterlihatan status sistem (visibility of system status) di mana informasi esensial berupa ringkasan pendapatan harian dan visualisasi tren penjualan disajikan secara seketika (real-time). Struktur navigasi diletakkan secara vertikal di sebelah kiri (sidebar) untuk mempertahankan konteks posisi pengguna secara konsisten dalam struktur arsitektur informasi aplikasi POS PWA.

IDKomponenFungsiInteraksi
A1Sidebar NavigasiMenu utama aplikasi. Logo & nama toko di header, profil di footer. Sidebar selalu expanded.Klik item → navigasi halaman
B1–B4Kartu Statistik (StatsCard)Omzet Hari Ini, Total Transaksi, Rata-rata/Transaksi, Menu Terlaris. Data real dari server, khusus hari ini.Tampilan pasif, data live dari TRPC query
C1Grafik Pendapatan (RevenueChart)Grafik batang tren pendapatan harian dalam periode tertentuDropdown filter → ganti periode
D1Penjualan Menu Teratas (TopProducts)Nama produk + qty porsi di kanan. Progress bar horizontal: 100% = produk terlaris. Maks 5 produk.Pasif

Keterangan Komponen — WF-01-M (Mobile)

Adaptasi antarmuka Dashboard pada layar perangkat seluler (mobile) difokuskan pada paradigma mobile-first dan pola desain ramah sentuhan (touch-friendly design). Pemadatan informasi numerik ke dalam bentuk matriks 2×2 menghindari kompleksitas visual berlebih pada area dengan keterbatasan spasial, sementara metode gulir lewati batas (horizontal scrolling) difasilitasi pada grafik batang agar integritas data tetap terjaga. Navigasi struktural ditransisikan ke pola Bottom Tab Bar agar memenuhi Hukum Fitts (Fitts's Law) untuk ketercapaian ibu jari secara ergonomis.

IDKomponenFungsiInteraksi
A1Header MobileLogo toko, nama halaman, ikon status koneksi PWA (online/offline)—
B1–B4StatsCard (2×2)Omzet Hari Ini, Total Transaksi, Rata-rata/Transaksi, Menu Terlaris. Grid 2 kolom untuk layar 375px.Pasif, data live TRPC
C1Grafik PendapatanGrafik batang dalam kontainer scrollable secara horizontalSwipe horizontal
D1Penjualan Menu TeratasNama produk + qty porsi. Progress bar horizontal relatif produk teratas. Maks 3 baris di mobile.Pasif
E1Bottom Tab BarNavigasi 5 tab. Dashboard aktif. Tab lain: Kasir, Produk, Laporan, Akun. Tab ke-5 membuka Account Bottom Sheet (logon/logout/profil).Tap tab → pindah halaman; Tap Akun → buka sheet
