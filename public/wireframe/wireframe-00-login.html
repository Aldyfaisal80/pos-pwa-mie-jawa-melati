Keterangan — WF-00-D

Antarmuka Login Desktop mengimplementasikan pola centered card di atas latar berwarna gelap, menciptakan focal point tunggal yang memandu perhatian pengguna langsung ke formulir autentikasi. Desain ini mengikuti prinsip progressive disclosure, di mana hanya elemen yang relevan dengan tugas autentikasi yang ditampilkan, mengeliminasi distraksi navigasional. Tidak ada sidebar atau navigasi utama karena pengguna belum terotentikasi. Setelah login berhasil, aplikasi melakukan redirect ke halaman Dashboard sebagai halaman utama pascaautentikasi.

IDKomponenFungsiInteraksi
A1Logo Toko (circle)Ikon identitas aplikasi ditampilkan di bagian atas kartu login. Membangun kepercayaan brand dan konteks visual bahwa ini adalah halaman masuk POS.Tidak ada interaksi — ornamental
A2Nama & Subjudul AplikasiLabel "Mie Jawa POS" dan teks "Masuk untuk melanjutkan" sebagai identifikasi aplikasi dan instruksi singkat bagi pengguna.Tidak ada interaksi — deklaratif
B1Input EmailField email pengguna terdaftar. Terhubung ke Supabase Auth. Placeholder: "admin@toko.com". Validasi format email dilakukan sisi klien sebelum request dikirim.Input → validasi format → submit
B2Input PasswordField password tersembunyi (type="password"). Karakter diganti dengan ikon •. Validasi: tidak boleh kosong.Input → submit; tombol show/hide opsional
C1Tombol MasukTombol submit utama yang mengirimkan kredensial ke Supabase Auth via signInWithPassword(). Menampilkan loading state saat proses berlangsung.Klik → loading → redirect Dashboard / tampil error
D1Footer CopyrightTeks hak cipta aplikasi di bagian bawah kartu. Memberikan informasi kepemilikan dan tahun rilis.Tidak ada interaksi — legal text

Keterangan — WF-00-M

Tampilan Login Mobile memanfaatkan desain yang identik dengan versi desktop karena komponen kartu login (centered card) bersifat responsif dan sudah mobile-first secara alami. Tidak ada perbedaan struktural atau navigasional antara tampilan desktop dan mobile pada halaman ini, karena halaman login tidak memiliki elemen navigasi yang perlu disesuaikan. Ukuran tipografi dan proporsi kartu disesuaikan secara proporsional terhadap konstrain layar perangkat genggam.

IDKomponenFungsiInteraksi
A1Login Card (Responsif)Kartu login menggunakan lebar penuh kontainer dengan padding internal. Sama persis dengan desktop karena tidak butuh adaptasi layout besar.Vertikal scroll jika keyboard muncul
B1Input Email & PasswordForm input standar, dioptimalkan untuk keyboard mobile (type="email" memunculkan keyboard @ otomatis, type="password" menyembunyikan karakter).Tap → keyboard muncul → input
C1Tombol MasukFull-width button yang mudah dijangkau ibu jari. Background gelap dengan label putih untuk visibilitas maksimal sebagai CTA (Call to Action) utama.Tap → loading → redirect / error

Keterangan — WF-00-E

State Error Login menampilkan umpan balik negatif ketika kredensial yang dimasukkan tidak valid. Pesan error muncul di atas form sebagai alert banner berwarna merah muda (bg-red-50 dengan border merah), mengikuti konvensi UI desain sistem yang membedakan error level regional (di dalam form) dari error level global (toast). Bidang input yang gagal mendapat border merah sebagai penanda visual tambahan.

IDKomponenFungsiInteraksi
E1Error Alert BannerBanner merah di atas form menampilkan pesan "Email atau password salah". Muncul setelah Supabase Auth mengembalikan error AuthApiError.Auto-tampil saat auth gagal; hilang saat form diisi ulang
E2Input Error StateBorder input berubah menjadi merah (border-red-300) sebagai visual cue tambahan bahwa data yang dimasukkan tidak valid.Error state reset saat user mulai mengetik ulang
E3Tombol Masuk (aktif)Tombol tetap aktif pada error state, memungkinkan pengguna untuk mengisi ulang dan mencoba login kembali tanpa refresh halaman.Klik ulang → kirim request baru ke Supabase Auth
